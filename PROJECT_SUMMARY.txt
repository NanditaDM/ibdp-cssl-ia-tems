=== OOP CONCEPTS IMPLEMENTED ===

1. Encapsulation
   - All class fields are private (#fieldName)
   - Public getter/setter methods for controlled access
   - Application class encapsulates all data arrays

2. Inheritance
   - User hierarchy: User -> Employee -> Manager
   - User -> HR, IT, CEO
   - Record hierarchy: Record -> all record types
   - super() calls to parent constructors

3. Polymorphism
   - Different user types handled through instanceof checks
   - Different behaviors based on user type
   - Method overriding in constructors

4. Libraries of Objects
   - Arrays storing collections of users and records
   - users[], requests[], tickets[], reports[], etc.

5. Access Modifiers
   - Private fields (# prefix)
   - Public methods
   - Protected behavior through class structure

- Send notifications to team/individuals

HR Features:
- View all behavioral reports

IT Features:

CEO Features:
